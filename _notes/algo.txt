1. В таск-трекере создаем и оформляем задачу
2. Создаем ветку для решения задачи
3. Локально переключаемся на ветку задачи
4. В папке для хранения решений задач (js) создаем папку для решения конкретной задачи taskName
5. В папке для решения задачи создаем файл для решения задачи taskName.js
6. В папке для решения задачи создаем файл для описания задачи README.md
7. В папке для решения задачи создаем файл для типов (опционально) types.ts
8. В папке для решения задачи создаем файл для тестов задачи - taskName.test.js
9. Заполняем файл README.md
10. Заполняем файл для тестов
11. Заполняем файл для типов
12. В файл для решения задачи импортируем типы из файла типов с помощью JSDoc комментария
13. Описываем мини документацию функции (решение задачи) с помощью JSDoc комментария
14. Определяем функцию (решение задачи)
15. Объявляем параметры функции (опионально)
16. Обдумываем условие задачи
17. Собираем всю необходимую информацию в конспект для решения задачи (GPT)(опионально)
18. В теле функции в виде комментариев, построчно описываем алгоритм решения задачи (псевдокод)
19. Каждый элемент алгоритма на псевдо-коде заменяем синтаксической инструкцией (пишем на языке программирования)
20. Запускаем тест (прогоняем через тест решение задачи)
21. Если тест пройден и отсутствует замечания линтеров (линтер типов и линтер синтаксиса), то задача решена
22. Коммиты и PR на проверку
23. Если Aprowe, то merge и закрытие задачи.
24. Обновляем локальную главную ветку
25. Переходим к решению следующей задачи
